# Citation Generation Skill

## Overview

The **Citation Generation Skill** automatically identifies, formats, and appends source references to answers generated by the RAG chatbot. By cross-referencing each segment of a generated answer with its originating document chunk, this skill produces inline citations, footnotes, or bibliographic entries in standard academic and professional formats (APA, MLA, IEEE, Chicago). It transforms the chatbot from a black-box responder into a **transparent, auditable, and academically rigorous** information system.

---

## Purpose

Trustworthiness is one of the greatest challenges in deploying AI-powered chatbots in professional, academic, and healthcare settings. The Citation Generation Skill directly addresses this by ensuring every factual claim in the chatbot's answer can be traced back to a specific passage in a specific source document. This is critical for compliance, fact-checking, academic submission, and professional accountability. It also enables users to independently verify information and explore source documents for deeper reading.

---

## Inputs

| Input | Type | Description |
|-------|------|-------------|
| `generated_answer` | `str` | The answer produced by the Question Answering Skill |
| `retrieved_chunks` | `List[str]` | The document chunks used to generate the answer |
| `source_metadata` | `List[dict]` | Metadata per chunk: filename, author, page, date, URL |
| `citation_format` | `str` | Output format: `"APA"`, `"MLA"`, `"IEEE"`, `"Chicago"`, `"inline"` |
| `include_inline` | `bool` | Whether to embed inline citation markers `[1]` within the answer text |

---

## Outputs

| Output | Type | Description |
|--------|------|-------------|
| `cited_answer` | `str` | The original answer with inline citation markers inserted |
| `references_list` | `List[str]` | Formatted bibliography entries in the requested citation style |
| `citation_map` | `dict` | Mapping of citation numbers to source chunk and document metadata |

---

## Workflow

1. **Chunk-Answer Alignment** — The skill tokenizes the generated answer into sentences and computes semantic similarity between each sentence and each retrieved chunk to identify which chunk best supports each claim.
2. **Citation Marker Insertion** — Sentence-level citation markers (e.g., `[1]`, `[2]`) are inserted at the end of each factual sentence in the answer.
3. **Deduplication** — If multiple sentences are supported by the same source chunk, a single citation marker is reused to avoid redundant references.
4. **Metadata Extraction** — Author names, publication year, document title, page numbers, and URLs are extracted from the source metadata dictionary.
5. **Format Rendering** — Bibliographic entries are rendered in the requested citation style (APA, MLA, IEEE, etc.) using the extracted metadata fields.
6. **Reference List Assembly** — All unique references are numbered, sorted, and assembled into a formatted reference list.
7. **Output Packaging** — The cited answer, reference list, and citation map are returned as a structured object.

## Example Input

```python
generated_answer = "Transformer models process sequences in parallel using self-attention mechanisms. BERT introduced bidirectional pre-training, significantly improving performance on NLP benchmarks. GPT models use decoder-only architecture for autoregressive text generation."

source_metadata = [
    {"title": "Attention is All You Need", "authors": "Vaswani et al.", "year": 2017, "page": 3},
    {"title": "BERT: Pre-training of Deep Bidirectional Transformers", "authors": "Devlin et al.", "year": 2019, "page": 1},
    {"title": "Language Models are Few-Shot Learners", "authors": "Brown et al.", "year": 2020, "page": 6}
]

citation_format = "APA"
include_inline = True
```

## Use Cases

1. **Academic Research Chatbot** — Automatically cite retrieved journal articles and textbooks in APA or IEEE format for student and researcher use.
2. **Legal Research Platform** — Generate citations to specific clauses, case references, and statutory provisions within legal document searches.
3. **Medical Information System** — Link every clinical recommendation to its evidence source (e.g., PubMed article, clinical guideline) for physician trust.
4. **Journalism Support Tool** — Auto-cite sources for AI-assisted news drafting tools to uphold journalistic standards and attribution.
5. **Enterprise Knowledge Management** — Cite internal SOPs, policy documents, and technical manuals when answering employee queries.
6. **Educational Assessment** — Provide students with cited study materials that train them in referencing conventions alongside content delivery.

---

## Benefits

- **Accountability** — Every factual claim is traceable, making the chatbot's outputs verifiable and auditable.
- **Trust Building** — Users are more likely to trust and act on information that comes with transparent source attribution.
- **Academic Compliance** — Enables chatbot outputs to be directly used in academic and professional reports with minimal reformatting.
- **Hallucination Detection** — Inline citation mapping makes it easier for reviewers to identify unsupported claims in the generated answer.
- **Format Versatility** — Multi-format support (APA, MLA, IEEE) makes the skill adaptable across disciplines and institutions.

---

## Future Enhancements

- **DOI & URL Resolution** — Automatically append DOI links or URLs to bibliographic entries for digital accessibility.
- **Citation Completeness Checker** — Flag answers where sentences lack supporting citations, prompting further retrieval.
- **Footnote Mode** — Support footnote-style citations for document export (e.g., Word, PDF) in addition to in-text markers.
- **Citation Clustering** — Group related citations into composite references (e.g., `[1,3,5]`) to reduce inline clutter.
- **Style Auto-Detection** — Detect the appropriate citation style based on the domain (science → IEEE, humanities → MLA).
